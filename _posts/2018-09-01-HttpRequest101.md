네트워크 요청을 일반적으로 백그라운드에서 수행된다. 그 이유는 네트워크 작업 특성상 응답이 올 때까지 어느 정도 시간이 걸리기 때문이다. 

iOS의 모든 화면(UI) 갱신과 관련된 작업들은 메인 스레드(main thread)에서 수행된다. 네트워크 관련 코드를 메인 스레드에서 수행되도록 코드를 실행하면, 네트워크 응답이 완료되기 전까지 사용자는 UI가 멈춘 것 처럼 느낄 것이다. 이는 응답이 없는 오류이며 나쁜 사용자 경험으로 볼 수 있다.

> 같은 맥락에서 유저인터페이스(UI) 요소를 갱신할 때는 반드시 메인큐에서 수행되도록 해야 한다.

이와 같은 이유로 모든 네트워크 작업은 백그라운드 큐에서 호출 해야 한다. 다만 결과는 메인 큐에서 보여주어야 한다. GCD를 사용한 예제 코드(의사코드)는 다음과 같다.

```Swift
DispatchQueue.global(qos: DispatchQoS.QoSClass.background).async(execute: {
   doNetworkingOperation { (results) in
       DispatchQueue.main.async(execute: { showResultsInMainQueue(results) })
   }
})
```
